🌍 Аналогія з реального життя

Перш ніж писати код, давай розберемось через звичайні речі навколо нас. Це допоможе зрозуміти різницю між звичайним і статичним класом на інтуїтивному рівні.

💡

Уявіть: ви відвідуєте бібліотеку. У ній є дві принципово різні речі — книги-шаблони (за якими друкують нові копії) та стаціонарні довідники (вони завжди на місці, їх не позичають).

📖

Звичайний клас
(Regular Class)

Як шаблон книги — з нього можна надрукувати багато екземплярів. Кожна книга (об'єкт) має власний вміст, закладки, нотатки власника. Усі вони незалежні одна від одної.

📚

Статичний клас
(Static Class)

Як великий довідник у читальному залі — він один і єдиний, прикутий ланцюжком до столу. Ніхто не може забрати його копію. Ти просто підходиш і користуєшся.

🏭

Ще аналогія:
Завод vs Стандарти

Завод (звичайний клас) виробляє окремі автомобілі. Кожен авто — свій об'єкт з власним кольором, пробігом, власником.

📏

Правила дорожнього
руху = статика

ПДР (статичний клас) — один для всіх. Немає «мого примірника ПДР» — є одні спільні правила, якими всі користуються однаково.

🎯

Коротко: звичайний клас — це шаблон для створення об'єктів. Статичний клас — це набір інструментів або правил, якими ти користуєшся напряму, без створення об'єктів.

📦 Що таке статичний клас?

У C# статичний клас — це клас, позначений ключовим словом static. Він принципово відрізняється від звичайного: з нього неможливо створити об'єкт.

C# 
// ✅ Звичайний клас — можна створити об'єкт
public class Dog
{
    public string Name { get; set; }
    public void Bark() => Console.WriteLine("Гав!");
}

// ✅ Статичний клас — ТІЛЬКИ статичні члени
public static class MathHelper
{
    public static double Square(double x) => x * x;
    public static bool IsEven(int n) => n % 2 == 0;
}

// Як використовуємо:
Dog myDog = new Dog();        // ✅ Можна!
myDog.Bark();

// MathHelper helper = new MathHelper(); // ❌ ПОМИЛКА!
double result = MathHelper.Square(5); // ✅ Ось так правильно
Console.WriteLine(result); // 25
💬

Зверни увагу: для звичайного класу ми пишемо new Dog(), а для статичного звертаємось напряму за іменем класу: MathHelper.Square(). Нічого зайвого!

🧩 Де статичні класи зустрічаються в реальному C#?

До речі, ми вже використовували статичні класи, навіть не знаючи про це!

C# 
// Console — статичний клас!
Console.WriteLine("Привіт, світе!"); // Не пишемо new Console()

// Math — статичний клас!
double root = Math.Sqrt(16);  // Результат: 4
double pi   = Math.PI;        // 3.14159...
int    abs  = Math.Abs(-42); // Результат: 42

// Convert — статичний клас!
int num = Convert.ToInt32("123"); // "123" → 123
string s = Convert.ToString(42);  // 42 → "42"

// Array — статичний клас для роботи з масивами!
int[] nums = { 3, 1, 4, 1, 5 };
Array.Sort(nums);  // { 1, 1, 3, 4, 5 }
🎉

Бачиш? Math, Console, Convert, Array — все це статичні класи!

⚙️ Особливості та правила статичних класів

Статичний клас — не просто клас зі словом static. Він має чіткі правила. Давай розберемо кожне з них детально.


🚫 Правило 1 — Не можна створити екземпляр ▼

Ключове слово static буквально «забороняє» компілятору C# дозволяти new для цього класу. Якщо спробуєш — отримаєш помилку ще до запуску програми!

C# 
public static class Calculator
{
    public static int Add(int a, int b) => a + b;
}

// ❌ Помилка компілятора:
// Cannot create an instance of the static class 'Calculator'
Calculator calc = new Calculator(); // ❌

// ✅ Правильний виклик:
int sum = Calculator.Add(10, 5); // 15
🔒 Правило 2 — Всі члени мають бути статичними ▼

В середині статичного класу кожне поле, метод, властивість — усе має мати ключове слово static. Без винятків!

C# 
public static class AppConfig
{
    // ✅ Статичне поле
    public static string AppName = "Мій Застосунок";

    // ✅ Статична константа
    public static readonly int Version = 1;

    // ✅ Статична властивість
    public static string Author { get; set; } = "Іван";

    // ✅ Статичний метод
    public static void PrintInfo()
    {
        Console.WriteLine($"Застосунок: {AppName} v{Version}");
    }

    // ❌ Помилка! Нестатичний метод в статичному класі:
    // public void SomeMethod() { } // Compile error!
}
🚫 Правило 3 — Не можна успадковувати ▼

Статичний клас автоматично стає запечатаним (sealed). Ні від нього не можна успадкуватись, ні він сам не може успадковувати інший клас (крім object).

C# 
public static class UtilityBase
{
    public static void DoSomething() { }
}

// ❌ Помилка: не можна успадкуватись від статичного класу
// public class MyUtils : UtilityBase { } // Compile error!

// ❌ Помилка: статичний клас не може успадкуватись від іншого класу
// public static class MyStatic : SomeClass { } // Compile error!

// ✅ Статичний клас може реалізовувати інтерфейси через extension methods
// Але не може прямо вказати: StaticClass : IInterface
🧠

Чому? Уяви, що є правило «1 унікальний клас». Якщо б можна було успадковувати, з'явились би «копії» з небажаними змінами. Статичний клас — це синглтон-утиліта, а не ієрархія.

💾 Правило 4 — Існує весь час роботи програми ▼

Статичний клас завантажується в пам'ять один раз при першому використанні і живе доти, доки програма не завершиться. Не потрібно думати про його «видалення».

C# 
public static class VisitCounter
{
    // Ця змінна живе весь час роботи програми
    public static int Count = 0;

    public static void Visit()
    {
        Count++;
        Console.WriteLine($"Відвідувань: {Count}");
    }
}

// В різних місцях програми:
VisitCounter.Visit(); // Відвідувань: 1
VisitCounter.Visit(); // Відвідувань: 2
VisitCounter.Visit(); // Відвідувань: 3
// Значення зберігається між всіма викликами!

🏗️ Звичайний клас — «Шаблон для об'єктів»

Використовується для моделювання сутностей реального світу, що мають стан (поля) і поведінку (методи).

C# 
public class BankAccount
{
    // Власний стан кожного об'єкту
    public string Owner { get; set; }
    public decimal Balance { get; private set; }

    public BankAccount(string owner)
    {
        Owner = owner;
        Balance = 0;
    }

    public void Deposit(decimal amount) => Balance += amount;
    public void Withdraw(decimal amount) => Balance -= amount;
}

// Кожен — окремий об'єкт зі своїм станом!
BankAccount alice = new BankAccount("Аліса");
BankAccount bob   = new BankAccount("Боб");

alice.Deposit(1000); // Аліса: 1000
bob.Deposit(500);   // Боб: 500 — незалежно!

🎭 Абстрактний клас — «Незавершений шаблон»

Схожий на звичайний, але призначений тільки для успадкування. Може містити абстрактні методи — «обіцянки», що нащадки їх реалізують.

C# 
// Абстрактний клас — не можна створити напряму
public abstract class Shape
{
    public string Color { get; set; } = "Червоний";

    // Абстрактний метод — нащадок ЗОБОВ'ЯЗАНИЙ реалізувати
    public abstract double GetArea();

    // Звичайний метод — є реалізація
    public void Describe()
    {
        Console.WriteLine($"{Color} фігура, площа: {GetArea():F2}");
    }
}

// ✅ Нащадки реалізують GetArea по-своєму
public class Circle : Shape
{
    public double Radius { get; set; }
    public override double GetArea() => Math.PI * Radius * Radius;
}

public class Rectangle : Shape
{
    public double Width, Height;
    public override double GetArea() => Width * Height;
}

// Shape s = new Shape(); // ❌ Помилка!
Circle c = new Circle { Radius = 5, Color = "Синій" };
c.Describe(); // Синій фігура, площа: 78.54

⚡ Статичний клас — «Набір інструментів»

Не для об'єктів, не для успадкування — тільки для утиліт і допоміжних методів, якими всі користуються однаково.

C# 
public static class StringUtils
{
    // Перевірка — email чи ні?
    public static bool IsEmail(string s)
        => s.Contains('@') && s.Contains('.');

    // Скорочення рядка
    public static string Truncate(string s, int maxLen)
        => s.Length <= maxLen ? s : s[..maxLen] + "...";

    // Рахуємо слова
    public static int WordCount(string s)
        => s.Split(' ', StringSplitOptions.RemoveEmptyEntries).Length;
}

// Використання — просто і ясно:
Console.WriteLine(StringUtils.IsEmail("test@mail.com")); // True
Console.WriteLine(StringUtils.Truncate("Довгий рядок тексту", 10)); // Довгий ра...
Console.WriteLine(StringUtils.WordCount("Привіт мій друже"));  // 3
🧠

Головне правило вибору: Якщо тобі потрібно моделювати об'єкти зі станом → звичайний або абстрактний клас. Якщо тобі потрібен набір функцій без стану кожного об'єкту → статичний клас.

🔧 Статичні конструктори

Конструктор — це спеціальний метод, що запускається при «народженні» об'єкту. А що таке статичний конструктор?

🎯

Статичний конструктор — це метод, який запускається автоматично один раз, коли клас вперше використовується в програмі. Він призначений для ініціалізації статичних даних.

📋 Синтаксис статичного конструктора

C# 
public static class AppSettings
{
    public static string ServerUrl;
    public static int TimeoutMs;

    // ✅ Статичний конструктор:
    // - Без модифікатора доступу (не public/private)
    // - Без параметрів
    // - Ключове слово static
    static AppSettings()
    {
        ServerUrl = "https://api.myapp.com";
        TimeoutMs = 5000;
        Console.WriteLine("⚙️ AppSettings ініціалізовано!");
    }
}

// Перший виклик — статичний конструктор запустився!
Console.WriteLine(AppSettings.ServerUrl);
// Вивід: ⚙️ AppSettings ініціалізовано!
// Вивід: https://api.myapp.com

Console.WriteLine(AppSettings.TimeoutMs);
// Вивід: 5000 (конструктор вже не запускається знову!)

🔑 Ключові особливості статичного конструктора

  • Запускається автоматично — ти не можеш викликати його напряму. CLR (середовище виконання C#) сам вирішує, коли це зробити.
  • Запускається лише один раз — незалежно від того, скільки разів використовується клас.
  • Без параметрів — на відміну від звичайних конструкторів, статичний не приймає аргументів.
  • Без модифікатора доступу — не пишемо public, private тощо. Просто static ClassName().
  • Перший — завжди статичний — якщо в класі є і статичний, і звичайний конструктор, статичний завжди запускається першим.

C# 
public class Database
{
    private static string _connectionString;
    public string Name { get; }

    // Статичний конструктор — викликається ПЕРШИМ, ОДИН РАЗ
    static Database()
    {
        _connectionString = "Server=localhost;Port=5432";
        Console.WriteLine("🔌 Рядок підключення ініціалізовано");
    }

    // Звичайний конструктор — викликається при кожному new
    public Database(string name)
    {
        Name = name;
        Console.WriteLine($"📦 Створено підключення до: {Name}");
    }
}

// Демонстрація порядку:
Database db1 = new Database("ProductsDB");
// Вивід: 🔌 Рядок підключення ініціалізовано
// Вивід: 📦 Створено підключення до: ProductsDB

Database db2 = new Database("UsersDB");
// Вивід: 📦 Створено підключення до: UsersDB
// (статичний конструктор вже не повторюється!)
⚠️

Важливо! Якщо в статичному конструкторі станеться помилка (виняток), клас стане «недоступним» на весь час роботи програми. Тому ініціалізацію в ньому роблять простою і надійною.

💡 Практичні приклади

Тепер час побачити статичні класи в дії. Ось реальні сценарії, де вони використовуються!


🔌 Приклад 1 — Методи розширення (Extension Methods)

Один з найпотужніших паттернів C# — додавати методи до існуючих типів без зміни їх коду!

C# 
// Статичний клас для методів розширення
public static class StringExtensions
{
    // "this string s" — означає, що це розширення типу string
    public static bool IsEmail(this string s)
        => s.Contains('@') && s.Contains('.');

    public static string ToPigLatin(this string s)
        => s + "-хрю"; // жартівливий приклад 🐷

    public static string Repeat(this string s, int times)
        => string.Concat(Enumerable.Repeat(s, times));
}

public static class IntExtensions
{
    public static bool IsPrime(this int n)
    {
        if (n < 2) return false;
        for (int i = 2; i * i <= n; i++)
            if (n % i == 0) return false;
        return true;
    }

    public static int Factorial(this int n)
        => n <= 1 ? 1 : n * (n - 1).Factorial();
}

// Тепер можна викликати методи прямо на значеннях!
string email = "user@example.com";
Console.WriteLine(email.IsEmail()); // True
Console.WriteLine("Ха".Repeat(3));   // ХаХаХа
Console.WriteLine(7.IsPrime());    // True
Console.WriteLine(5.Factorial());  // 120

⚙️ Приклад 2 — Конфігурація застосунку зі статичним конструктором

Статичний конструктор — ідеальне місце для завантаження налаштувань при старті програми.

C# 
public static class GameConfig
{
    public static int    MaxLevel    { get; }
    public static int    StartLives  { get; }
    public static string GameName    { get; }
    public static bool   SoundEnabled{ get; }

    // Статичний конструктор — запускається ОДИН РАЗ на старті
    static GameConfig()
    {
        Console.WriteLine("🎮 Завантаження конфігурації гри...");

        // Зазвичай тут читаємо файл або змінні середовища
        // Для прикладу — захардкодимо:
        MaxLevel     = 50;
        StartLives   = 3;
        GameName     = "Зоряний Шлях";
        SoundEnabled = true;

        Console.WriteLine($"✅ Гра '{GameName}' готова! Рівнів: {MaxLevel}");
    }

    public static void PrintConfig()
    {
        Console.WriteLine($"🎮 {GameName}");
        Console.WriteLine($"   Рівнів: {MaxLevel}");
        Console.WriteLine($"   Стартові життя: {StartLives}");
        Console.WriteLine($"   Звук: {(SoundEnabled ? "✅" : "❌")}");
    }
}

// Перший доступ — конструктор запускається
GameConfig.PrintConfig();
// Другий доступ — конструктор вже не повторюється
Console.WriteLine(GameConfig.MaxLevel);

📝 Приклад 3 — Логер (Logger)

Статичний клас ідеально підходить для логування — нам не потрібні окремі екземпляри, лише єдиний запис подій.

C# 
public static class Logger
{
    private static List<string> _logs = new();
    private static int _errorCount = 0;

    static Logger()
    {
        Console.WriteLine("📋 Логер запущено: " + DateTime.Now);
    }

    public static void Info(string message)
    {
        string log = $"[{DateTime.Now:HH:mm:ss}] ℹ️ INFO: {message}";
        _logs.Add(log);
        Console.WriteLine(log);
    }

    public static void Error(string message)
    {
        _errorCount++;
        string log = $"[{DateTime.Now:HH:mm:ss}] ❌ ERROR: {message}";
        _logs.Add(log);
        Console.WriteLine(log);
    }

    public static void Success(string message)
    {
        string log = $"[{DateTime.Now:HH:mm:ss}] ✅ OK: {message}";
        _logs.Add(log);
        Console.WriteLine(log);
    }

    public static void Summary()
    {
        Console.WriteLine($"\n📊 Зведення: {_logs.Count} подій, {_errorCount} помилок");
    }
}

// Використовуємо по всій програмі:
Logger.Info("Програма запущена");

Logger.Error("Не вдалось підключитись!");

🔢 Приклад 4 — Розширені математичні утиліти

Класичний use-case: допоміжні математичні функції, яких немає в стандартному Math.

C# 
public static class AdvancedMath
{
    // Число Фібоначчі
    public static long Fibonacci(int n)
    {
        if (n <= 1) return n;
        long a = 0, b = 1;
        for (int i = 2; i <= n; i++)
            (a, b) = (b, a + b);
        return b;
    }

    // Найбільший спільний дільник
    public static int GCD(int a, int b)
        => b == 0 ? a : GCD(b, a % b);

    // Чи є число степенем двійки?
    public static bool IsPowerOfTwo(int n)
        => n > 0 && (n & (n - 1)) == 0;

    // Відсоток від числа
    public static double Percent(double value, double percent)
        => value * percent / 100;

    // Обмеження значення в діапазоні
    public static T Clamp<T>(T value, T min, T max) where T : IComparable<T>
    {
        if (value.CompareTo(min) < 0) return min;
        if (value.CompareTo(max) > 0) return max;
        return value;
    }
}

// Тестуємо:
Console.WriteLine(AdvancedMath.Fibonacci(10));     // 55
Console.WriteLine(AdvancedMath.GCD(48, 18));         // 6
Console.WriteLine(AdvancedMath.IsPowerOfTwo(64));    // True
Console.WriteLine(AdvancedMath.Percent(200, 15));    // 30
Console.WriteLine(AdvancedMath.Clamp(150, 0, 100));  // 100

✅ Приклад 5 — Валідатор даних

Перевірка даних — чудовий кандидат для статичного класу. Не потрібен стан, лише чисті функції.

C# 
public static class Validator
{
    public static bool IsEmail(string email)
        => !string.IsNullOrEmpty(email)
           && email.Contains('@')
           && email.Contains('.')
           && email.Length > 5;

    public static bool IsPhone(string phone)
        => phone.Length >= 10
           && phone.All(c => char.IsDigit(c) || c == '+' || c == '-');

    public static bool IsStrongPassword(string pwd)
        => pwd.Length >= 8
           && pwd.Any(char.IsUpper)
           && pwd.Any(char.IsDigit)
           && pwd.Any(char.IsLower);

    public static bool IsAge(int age)
        => age >= 0 && age <= 150;

    // Зручна обгортка для виводу результату
    public static void Check(string label, bool result)
        => Console.WriteLine($"{(result ? "✅" : "❌")} {label}: {result}");
}

// Тестуємо валідатор:
Validator.Check("Email",    Validator.IsEmail("user@mail.com")); // ✅
Validator.Check("Email",    Validator.IsEmail("невалідний"));     // ❌
Validator.Check("Телефон", Validator.IsPhone("+380501234567"));  // ✅
Validator.Check("Пароль",  Validator.IsStrongPassword("Abc123!")); // ✅
Validator.Check("Пароль",  Validator.IsStrongPassword("weak"));    // ❌
Validator.Check("Вік",     Validator.IsAge(25));               // ✅
Validator.Check("Вік",     Validator.IsAge(-5));               // ❌

🎯 Міні-тест — Перевір себе!

1 Яке ключове слово використовується для оголошення статичного класу?

🔵 abstract
🔵 static
🔵 sealed
🔵 readonly

2 Що станеться, якщо написати new MathHelper() для статичного класу?

🔵 Програма запустить конструктор і поверне порожній об'єкт
🔵 Програма впаде під час виконання
🔵 Компілятор видасть помилку ще до запуску
🔵 Нічого не станеться — код просто ігнорується

3 Скільки разів запускається статичний конструктор за час роботи програми?

🔵 Кожен раз при виклику будь-якого методу класу
🔵 Лише один раз — при першому звернення до класу
🔵 Двічі: на початку і в кінці програми
🔵 Залежить від кількості потоків

4 Який з наступних класів є прикладом статичного класу у стандартній бібліотеці C#?

🔵 List<T>
🔵 Exception
🔵 Math
🔵 StringBuilder

5 Чи може статичний клас мати нестатичні поля?

🔵 Так, якщо позначити їх як readonly
🔵 Так, тільки приватні поля
🔵 Ні, всі члени статичного класу мають бути статичними
🔵 Так, звичайні поля дозволені

6 Яка головна відмінність між абстрактним і статичним класом?

🔵 Абстрактний клас не може мати методів, а статичний може
🟢 Абстрактний клас призначений для успадкування, статичний — ні
🔵 Статичний клас дозволяє new, абстрактний — ні
🔵 Вони однакові, тільки різні ключові слова

📝 Підсумок

⚡ Статичний клас

  • Ключове слово: static class
  • Не можна: new, успадкування
  • Всі члени мають бути static
  • Живе весь час програми
  • Ідеальний для утиліт і хелперів

🔧 Статичний конструктор

  • Синтаксис: static ClassName()
  • Без параметрів і модифікаторів
  • Викликається автоматично один раз
  • Перший, якщо є звичайний конструктор
  • Для ініціалізації статичних даних

🏗️ Звичайний клас

  • Ключове слово: class
  • Можна: new, успадкування
  • Статичні і нестатичні члени
  • Кожен об'єкт — свій стан
  • Для моделювання сутностей

🎭 Абстрактний клас

  • Ключове слово: abstract class
  • Не можна: new напряму
  • Призначений для успадкування
  • Може мати абстрактні методи
  • Базовий шаблон для нащадків
